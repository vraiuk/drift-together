# «Сплав» — кооп-режим Drift Together (срез v0.2)

Дата: 2026-07-18. Основание: `Docs/UseCase_Splav_v0.1.md` (UC-01…UC-15).
Одобрено пользователем: сеть Unity Relay + код; объём «ядро сплава»;
режим внутри Drift Together; ходячие персонажи на плоту.

## Цель среза

2–4 друга соединяются по коду, вместе сплавляются на плоту по существующей
реке «Туманный лес» до финиша и получают общий «отчёт о сплаве». Соло-режим
остаётся без изменений.

Вошедшие use cases: UC-01 (сбор команды), UC-02 (руль), UC-03 (ремонт —
упрощённо), UC-05 (падение за борт/толкание, статус «мокрый»), UC-07
(кооп-гребля постами), UC-09 (упрощённо: «река возвращает» игрока),
UC-12 (развилка манёвром), UC-14 (стоянка-чекпоинт без межсессионного
сейва), UC-15 (финиш + отчёт с номинациями).

НЕ вошли (следующие итерации): рыбалка (UC-04), разведка (UC-06), переворот
(UC-08), полная потеря плота (UC-09), вылазки 3+1 и proximity voice (UC-10),
волок (UC-11), модули (UC-13), сейвы между запусками, водопад-сетпис.

## Сеть

- **Стек:** Netcode for GameObjects (com.unity.netcode.gameobjects) +
  Unity Transport + **Unity Relay** и **Lobby не используется** — код
  сессии = join-код Relay (6 символов), которым делятся в мессенджере.
- **UGS:** проект привязывается к Unity Gaming Services (Unity Cloud),
  сервисы Relay + Authentication (анонимная). Требуется один раз: вход
  пользователя в Unity Cloud и link project id (organization уже есть —
  аккаунт Unity Personal).
- **Авторитет:** хост авторитетен для плота, реки, прочности, чекпоинтов,
  статусов; персонажи — авторитет владельца (owner authority,
  ClientNetworkTransform-подход). Казуальный кооп друзей, античита нет.
- **Спавн-объекты:** все network-префабы собираются editor bootstrap'ом в
  `Assets/_Project/Prefabs` (плот, персонаж, network manager) и
  регистрируются в NetworkManager. Никакой ручной настройки Inspector.
- 2–4 игрока; подключение до старта сплава (в пути — reject с сообщением).
  Дисконнект гостя — персонаж исчезает, игра продолжается; дисконнект
  хоста — все возвращаются в меню с сообщением.

## Экраны и поток (UC-01)

1. Главное меню: новая кнопка **«Кооператив — Сплав»**.
2. Экран кооператива: «Создать сплав» / поле кода + «Войти» / «Назад».
3. Лобби хоста: список игроков (имя = «Игрок N» + цвет), крупный код
   сессии, для хоста выбор «С начала / С чекпоинта» (чекпоинт доступен,
   если в этой сессии процесса уже был сплав с чекпоинтом) и кнопка
   «Отчалить». Гости видят «ждём хоста».
4. «Отчалить» → сетевая загрузка сцены River в кооп-варианте, все
   просыпаются на плоту (спавн стоя на плоту).

## Плот (UC-02, UC-07)

- Прямоугольный плот ~4×3 м из брёвен (процедурные цилиндры), борта-упоры,
  мачта-флагшток с фонарём, костровая чаша в центре, Тапок-Тим сидит на
  бочке и комментирует (реплики выбирает хост, всем показываются одинаково).
- Физика — на хосте, та же аркадная модель, что у байдарки: течение реки
  толкает вниз, инерция, ограничение скорости; медленнее и тяжелее
  байдарки (масса выше, поворот ленивее).
- **Посты** (фиксированные точки, занять E, покинуть E/отойти):
  - **Руль** (корма): A/D — угол пера, задаёт доворот, эффективность
    растёт со скоростью.
  - **Весло левого борта / правого борта**: W — гребок вперёд со своего
    борта (тяга + доворот в противоположную сторону), S — табань.
    Гребки дискретные, с анимацией и брызгами; спам ограничен темпом.
  - Пост занят одним игроком; свободный пост может занять любой.
- Прочность плота **5**; жёсткие удары о камни/брёвна −1 с окном
  неуязвимости 1.5 с (переиспользуем HullIntegrity с maxPoints=5).
  При 0 — плот не тонет: скорость падает вдвое («расшатан»), нужен ремонт.
- **Ремонт (UC-03):** держать E у костровой чаши 3 с → +1 прочность,
  кулдаун 5 с. У пристани-стоянки — мгновенно до полной (см. чекпоинт).

## Персонажи (UC-05)

- Low-poly человечек (капсула + голова + кепка цвета игрока), третье лицо,
  камера своя у каждого (та же CameraRig, цель — свой персонаж).
- Управление: WASD — ходьба (по плоту — в локальных координатах плота,
  чтобы движение плота не сносило персонажа), E — взаимодействие
  (пост/костёр/залезть), F — **толкнуть** ближайшего игрока (импульс,
  кулдаун 3 с), Esc — пауза (локальная, игру не останавливает; сетевое
  время не замирает).
- **За бортом:** вода — плавание (медленно, к плоту), у борта — «E —
  залезть». Статус **«мокрый»** 60 с: −20% скорость ходьбы, капли;
  в радиусе костра сохнет в 4 раза быстрее. Смерти и урона игрокам нет.
- **Река возвращает (упрощённый UC-09):** если игрока отнесло дальше 40 м
  от плота или он вышел за пределы русла — телепорт к плоту (появляется
  у борта в воде, мокрый). Софтлок невозможен.

## Река, развилка, чекпоинт, финиш (UC-12, UC-14, UC-15)

- Уровень — существующая река «Туманный лес» с той же геометрией;
  в кооп-режиме грибы отключены, препятствия те же (для плота шире
  габарит — несколько камней у стен убираются кодом при кооп-сборке
  уровня).
- **Развилка:** решает фактический манёвр рулевого; при нерешительности
  течение само занесёт в один из рукавов (центрирующая сила выключена
  в зоне развилки — как в доке «течение выбирает само»). Выбранный рукав
  пишется в статистику.
- **Стоянка-чекпоинт:** существующая пристань с костром. Условие: плот в
  зоне, любой игрок жмёт E у берегового костра → checkpoint (позиция
  плота), полный ремонт, всем реплика Тима. «С чекпоинта» в лобби
  стартует плот с этой точки (в рамках сессии процесса хоста).
- **Финиш:** плот пересекает арку → 2 сек фанфары → всем экран
  **«Отчёт о сплаве»**: время, маршрут, столкновения плота, прочность на
  финише, падения за борт (по игрокам), суммарное «мокрое» время,
  номинации: «Главный по рулю» (больше всех у руля), «Мотор» (больше всех
  гребков), «Морж» (дольше всех мокрый), «Толкач» (больше всех толчков),
  «Сухарь» (ни разу за бортом; если таких нет — не выдаётся).
  Кнопки: «Ещё сплав» (хост; рестарт для всех) / «В меню».

## HUD

Прочность плота 5 делений, таймер, статус «мокрый» (иконка + прогресс),
контекстные подсказки («E — руль», «E — залезть», «F — толкнуть»),
субтитры Тима, код сессии в углу паузы. Все тексты русские.

## Архитектура кода

- `Scripts/Net`: `SessionManager` (создание/вход, Relay, join-код,
  жизненный цикл NGO), `NetPrefabs` (регистрация), `PlayerAvatar`
  (движение, посты, мокрый, толкание — owner authority),
  `RaftController` (физика плота, посты, прочность — host authority),
  `CoopFlow` (сетевой аналог GameFlow: состояния, чекпоинт, финиш,
  отчёт, статистика через NetworkVariable/RPC), `CoopStats`.
- Переиспользуются: RiverSpline/RiverFlow/LevelBuilder (+флаг кооп),
  HullIntegrity (параметризованный max), AudioSynth/AudioManager,
  UIBuilder/меню/субтитры/CameraRig, DialogueQueue/TimLines (+нужные
  категории), SmokeAutopilot (кооп-вариант для хоста).
- Editor bootstrap собирает network-префабы и сцену лобби не через
  Inspector, как и всё остальное.

## Тесты и проверки

- **Edit Mode:** прочность плота 5 + неуязвимость (параметризация
  HullIntegrity); логика постов (занять/освободить/один игрок на пост);
  таймер «мокрого» с ускорением у костра; подсчёт номинаций по статистике;
  выбор рукава по фактической позиции.
- **Play Mode:** NGO host+client в одном процессе (NetworkManager в
  host-режиме + виртуальный клиент): подключение, спавн двух аватаров,
  занятие поста, изменение NetworkVariable прочности. Полный smoke:
  хост-одиночка автопилотом проходит реку на плоту до отчёта.
- **Реальный smoke сборки:** `--smoke-coop` — хост без Relay (локальный
  host-mode), автопилот до финиша, `smoke_result.json`.
- Relay проверяется вручную одним запуском двух локальных копий
  (host + join по коду) — отражается в BUILD_REPORT.
